The observed failure of the factor-based covariance estimator—specifically its high condition number and poor performance relative to shrinkage—is directly attributable to the **volatility clustering (GARCH(1,1))** constraint described in the dataset.

The dataset description specifies that TSLA exhibits time-varying volatility with high persistence ($\beta=0.85$). The research plan relies on a 60-day rolling window to estimate idiosyncratic variances ($\Psi_t$). Because the data generating process includes GARCH-driven volatility, the idiosyncratic shocks are not homoskedastic over the 60-day estimation window. This creates a mismatch between the static assumption of the OLS-based factor model and the actual time-varying nature of the idiosyncratic noise. Consequently, the idiosyncratic variance estimates become highly unstable during volatility spikes, leading to the observed numerical instability (high condition number) and poor MVP performance.
